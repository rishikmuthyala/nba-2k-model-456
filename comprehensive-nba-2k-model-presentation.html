Introduction & Problem Statement

NBA 2K presents a complex decision environment with multiple variables impacting gameplay outcomes:

  • Player attributes (30+ variables per player)
  • Defensive positioning and pressure
  • Game context (score, time remaining, momentum)
  • Player stamina and fatigue effects
  • Shot clock and possession management

Current approaches rely primarily on intuition and experience, leading to:

  • Cognitive biases in high-pressure situations
  • Inconsistent decision-making
  • Sub-optimal outcomes in critical game moments

Core Research Question: Can we develop a mathematical model that quantifies optimal decision-making in NBA 2K to improve performance outcomes?

Shot Probability Model - Mathematical Formulation

P(shot) = B_r × F_{pos} × F_{def} × F_{fat} × F_{press}

Base Shot Rating (B_r)

B_r = \frac{PlayerRating}{100} × BaseConversionRate

Where PlayerRating is the appropriate attribute for the shot type (3PT, Mid-Range, Close) and BaseConversionRate is the baseline success rate for that shot type in NBA 2K.

Position Factor (F_pos)

F_{pos} = 1 - k_{pos} × d_{hotspot}

Where k_pos is a shot-specific decay constant and d_hotspot is the distance from the nearest player hotspot location.

Defender Factor (F_def)

F_{def} = 1 - \min(1, \frac{D_{rating}}{100} × \frac{1}{d_{def}^2 + \epsilon})

Where D_rating is the defender's defensive rating, d_def is the distance to defender, and ε is a small constant to prevent division by zero.

Fatigue & Pressure Factors

F_{fat} = 1 - k_{fat} × (1 - \frac{S_{current}}{S_{max}})

F_{press} = 1 - k_{press} × f(scoreDiff, timeRemaining)

Shot Model Technical Implementation

/** * Calculate shot success probability based on multiple factors * @param {Object} player - Player data with attributes * @param {Object} gameState - Current game state * @param {Object} shotContext - Shot-specific context * @return {Number} Probability between 0-1 */ function calculateShotProbability(player, gameState, shotContext) { // Get base rating based on shot type const baseRating = getBaseRating(player, shotContext.shotType); // Calculate position factor based on court location const positionFactor = calculatePositionFactor( shotContext.location, player.hotspots, MODEL_PARAMS.positionFactor.k_pos ); // Calculate defender factor const defenderFactor = calculateDefenderFactor( shotContext.defenderDistance, shotContext.defender.attributes.defense, MODEL_PARAMS.defenderFactor.epsilon ); // Calculate fatigue factor const fatigueFactor = calculateFatigueFactor( player.currentStamina, player.maxStamina, MODEL_PARAMS.fatigueFactor.k_fat ); // Calculate pressure factor const pressureFactor = calculatePressureFactor( gameState.scoreDifferential, gameState.timeRemaining, MODEL_PARAMS.pressureFactor.k_press ); // Combine all factors const probability = baseRating * positionFactor * defenderFactor * fatigueFactor * pressureFactor; // Ensure probability is within valid range return Math.max(0, Math.min(1, probability)); }

The model decomposes the shot success probability into individual factors that can be calibrated independently and then combined multiplicatively.

// Parameter Calibration Algorithm function calibrateModelParameters(shotData) { // Initialize parameter values let parameters = { baseConversionRates: { threePoint: 0.35, midRange: 0.45, closeShot: 0.65 }, positionFactor: { k_pos: 0.02 }, defenderFactor: { epsilon: 0.1 }, fatigueFactor: { k_fat: 0.25 }, pressureFactor: { k_press: 0.15 } }; // Define loss function (negative log likelihood) function lossFunction(params) { let logLikelihood = 0; shotData.forEach(shot => { const predictedProb = calculateShotProbability(shot, params); // Log likelihood for binary outcome logLikelihood += shot.result ? Math.log(predictedProb) : Math.log(1 - predictedProb); }); return -logLikelihood; } // Optimize parameters using gradient descent const optimizer = new GradientDescentOptimizer(lossFunction, parameters); const optimizedParams = optimizer.minimize(); // Validate parameters on hold-out data const validationError = validateParameters(optimizedParams, holdOutData); return { optimizedParams, validationError }; }

// Decision Tree Implementation function evaluateLatestGameDecision(gameState) { const { scoreDifferential, timeRemaining } = gameState; // Primary branch: Score Differential if (scoreDifferential <= -3) { // Trailing by 3+ if (timeRemaining < 24) { return "THREE_POINT_PRIORITY"; } else { return "HIGH_PPP_PLAY"; } } else if (scoreDifferential >= 3) { // Leading by 3+ if (timeRemaining < 35) { return "RUN_CLOCK_HIGH_PERCENTAGE"; } else { return "NORMAL_OFFENSE"; } } else { // Tied or within 2 if (timeRemaining < 35) { // Check for open shooter if (isBestShooterOpen(gameState)) { return "TAKE_SHOT"; } else { return "DRIVE_AND_KICK"; } } else { if (gameState.shotClock < 7) { return "TAKE_BEST_AVAILABLE_SHOT"; } else { return "RUN_OFFENSE_FOR_HIGH_PERCENTAGE"; } } } }

Expected Value Calculation

Mathematical Framework

EV(action) = \sum_{i=1}^{n} P(outcome_i) \times Value(outcome_i)

Shot Expected Value

EV(shot) = P(make) \times PointValue - P(miss) \times (1 - OReb\%) \times OppPPP

Where P(make) is the shot probability, PointValue is 2 or 3 points, OReb% is offensive rebound probability, and OppPPP is opponent's average points per possession.

Pass Expected Value

EV(pass) = P(complete) \times ReceivingEV - P(turnover) \times OppPPP

Where P(complete) is pass completion probability, ReceivingEV is expected value of receiver's position, and P(turnover) is turnover probability.

// Expected Value Implementation function calculateExpectedValue(action, gameState, playerData) { if (action.type === 'SHOT') { const shotProb = calculateShotProbability( playerData, gameState, action.context ); const pointValue = getShotPointValue(action.context.location); const offRebProb = calculateOffensiveReboundProbability(gameState); const oppPPP = gameState.opponent.pointsPerPossession; // Calculate expected value return (shotProb * pointValue) - ((1 - shotProb) * (1 - offRebProb) * oppPPP); } else if (action.type === 'PASS') { const passProb = calculatePassCompletionProbability( playerData, gameState, action.context ); const turnoverProb = 1 - passProb; const receivingEV = calculateReceiverExpectedValue( action.context.receiver, gameState ); return (passProb * receivingEV) - (turnoverProb * gameState.opponent.pointsPerPossession); } }

/** * Sensitivity Analysis for Model Parameters */ function performSensitivityAnalysis() { // Parameters to test const parameters = [ 'baseConversionRate.threePoint', 'baseConversionRate.midRange', 'positionFactor.k_pos', 'defenderFactor.epsilon', 'fatigueFactor.k_fat', 'pressureFactor.k_press' ]; // Variation ranges (percentage of base value) const variations = [-50, -25, -10, 0, 10, 25, 50]; // Results storage const results = {}; // Run sensitivity tests parameters.forEach(param => { results[param] = []; variations.forEach(variation => { // Create modified parameters const modifiedParams = { ...MODEL_PARAMS }; // Apply variation to target parameter const paramParts = param.split('.'); if (paramParts.length === 2) { const originalValue = modifiedParams[paramParts[0]][paramParts[1]]; modifiedParams[paramParts[0]][paramParts[1]] = originalValue * (1 + variation / 100); } // Evaluate model performance with modified parameters const performance = evaluateModelPerformance(modifiedParams); // Store results results[param].push({ variation, performance: { ppp: performance.pointsPerPossession, fg: performance.fieldGoalPercentage, win: performance.winPercentage } }); }); }); return results; }

Slide 19/24

Limitations of the Model

Technical Limitations

  • Parameter Calibration: Requires ongoing refinement for different player types
  • Simplified Mechanics: Some game mechanics are approximated
  • Defense Modeling: Less robust than offensive modeling
  • Data Constraints: Limited by sample size and data collection capabilities
  • Game Updates: May require recalibration after patches

Practical Limitations

  • Time Pressure: Difficult to apply full model during rapid gameplay
  • Implementation Gap: Knowing optimal decision ≠ execution ability
  • Cognitive Load: Requires mental bandwidth to apply in-game
  • Learning Curve: Understanding model logic takes time
  • Psychological Factors: Doesn't account for player psychology
